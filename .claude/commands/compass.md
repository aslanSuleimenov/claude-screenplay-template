Create an analytical compass for the project. Argument: $ARGUMENTS (genre, logline, or brief project description).

Ты создаёшь analytics/compass_artifact.md — главный аналитический справочник проекта. Он будет использоваться при написании и правке каждой сцены.

## Шаг 1: Сбор информации

Прочитай CLAUDE.md — жанр, персонажи, структура, стиль.

Затем поищи в интернете (используй WebSearch):
- Лучшие образцы жанра: какие фильмы/сериалы — эталоны? Почему работают? Включай русское и СНГ-кино наряду с мировым (например, Балабанов, Звягинцев, Быков, Лунгин, советская классика — если релевантны жанру)
- Жанровые конвенции: что зритель ожидает? Что нельзя нарушать?
- Структурные модели для этого жанра: трёхактная, пятиактная, эпизодическая?
- Драматические системы жанра: что двигает историю (страх? абсурд? моральный вопрос?)
- Антипримеры: что делает фильмы этого жанра плохими?

## Шаг 2: Составь compass_artifact.md

Файл должен содержать **4-8 жанровых систем**, каждая в формате:

```
## N. Название системы

Что это и почему важно для жанра. 2-3 абзаца с примерами из эталонных фильмов.

Закон системы: одна чёткая формулировка — правило, которому должна следовать каждая сцена.

Для [НАЗВАНИЕ ПРОЕКТА]: конкретное применение к данному сценарию.
```

Обязательные системы для большинства жанров:
- Крючок / открывающий инцидент
- Двигатель истории (центральный конфликт или заблуждение)
- Эскалация (как растут ставки)
- Структурная модель (с хронометражом петель/актов)
- Персонажные арки
- Тон и стиль (где граница жанра)
- Финал / батос / катарсис

## Шаг 3: Диагностика проекта

В конце файла добавь секцию **Диагностика**: что в текущем плане сценария уже работает, что под угрозой, что нужно добавить.

Сохрани в analytics/compass_artifact.md.

**Если файл уже существует** — перезапись допустима: жанр мог измениться в процессе работы. Перед перезаписью предупреди пользователя и покажи текущий жанр из существующего файла. После записи напомни: «Перезапусти /analyze на ключевых сценах — жанровые системы обновились.»
