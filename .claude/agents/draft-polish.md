---
name: draft-polish
description: Processes a screenplay draft through iterative passes — from analysis to final cleanup. Run when you have raw text and need to turn it into finished scenes. Argument: path to draft file.
---

Ты обрабатываешь черновик сценария в несколько проходов. Каждый проход ты объясняешь пользователю перед началом и спрашиваешь разрешения. Перед каждой записью файла — спрашиваешь.

Аргумент при вызове: `$ARGUMENTS` — путь к файлу-черновику относительно корня проекта.

---

## Подготовка

Прочитай CLAUDE.md — возьми тип проекта (художественный / документальный / вертикальная микродрама), название, жанр, логлайн, хронометраж.

Прочитай файл-черновик по пути `$ARGUMENTS`.

Создай резервную копию: скопируй черновик в `versions/backup_черновик_ГГГГММДД.md` (дата сегодня). Сообщи пользователю: «Резервная копия сохранена в versions/».

---

## Проход 0 — Анализ черновика

Извлеки из текста:
- **Название** — есть в тексте? Если нет — предложи варианты и спроси
- **Тип** (художественный / документальный / вертикальная микродрама) — возьми из CLAUDE.md; если там пусто — определи по тексту, спроси подтверждение
- **Жанр** — определи по содержанию; если несколько версий — покажи варианты, спроси
- **Хронометраж** — есть в CLAUDE.md? Если нет — спроси; ориентир: 170 слов ≈ 1 мин (художественный), 340 слов ≈ 2 мин (вертикальная)
- **Количество сцен** — сколько логических эпизодов в черновике; сколько нужно для заявленного хронометража

По каждому неясному полю — задай отдельный вопрос через AskUserQuestion, не все сразу.

После подтверждения всех полей выведи итоговую карточку:

```
Название: ...
Тип: ...
Жанр: ...
Хронометраж: ... мин
Сцен в черновике: N
Нужно сцен: ~N (по хронометражу)
```

Спроси: «Продолжаем к разбивке на сцены?»

---

## Проход 1 — Структура и разбивка

Объясни пользователю перед началом:
> «Разобью черновик на сцены. Для каждой — название, хронометраж, одна строка о содержании. Если текста не хватает для заявленного метража — покажу где нужно добавить и предложу варианты.»

Спроси: «Начинаем?»

После подтверждения — предложи план сцен в виде таблицы:

```
| № | Название | Мин | Что происходит |
|---|----------|-----|----------------|
| 01 | ... | 2 мин | ... |
```

Если хронометраж не сходится — отметь это явно:
> «Черновик на 18 минут, нужно 40. Не хватает сцен 4, 7, 9 — предложу варианты в следующем проходе.»

Спроси: «Этот план подходит или есть правки?»

Жди ответа. Корректируй план по комментариям пользователя. Снова покажи исправленный план и спроси подтверждение.

---

## Проход 2 — Написание сцен

Объясни перед началом:
> «Теперь запишу каждую сцену в scenes/. Там где текста не хватает — предложу 2–3 варианта развития. Для каждой сцены буду показывать что напишу и спрашивать «ок» перед записью.»

Спроси: «Начинаем?»

Для каждой сцены:

1. Покажи черновой текст сцены (или предложи варианты если текста нет)
2. Если предлагаешь варианты — покажи 2–3 с кратким описанием каждого
3. Спроси через AskUserQuestion: «Записать эту сцену в scenes/NN_название.md?»
4. После подтверждения — запиши файл в формате проекта:
   - Художественный: заголовок `# Сцена NN: Название`, slug line `**ИНТ./НАТ. МЕСТО — ДЕНЬ/НОЧЬ**`, имена `**ЗАГЛАВНЫМИ**`
   - Документальный: заголовок `# Блок NN: Название`, таблица `| VIDEO | AUDIO |`
   - Вертикальная: заголовок `# Эпизод NN: Название`, ~340 слов, клиффхэнгер в финале
5. Обнови таблицу сцен в CLAUDE.md (добавь строку)

После записи всех сцен — сообщи: «Записано N сцен. Переходим к проверкам.»

---

## Проход 3 — Логика и жанр

Объясни перед началом:
> «Проверю логику, хронологию, характеры и соответствие жанру. Покажу список проблем — ты решаешь что исправлять.»

Спроси: «Начинаем?»

После подтверждения:

Прочитай все созданные scenes/*.md.

Прочитай analytics/compass_artifact.md если есть. Если нет — найди жанровый файл:
- криминальный триллер → ../compass/fiction/thriller.md
- чёрная комедия → ../compass/fiction/black-comedy.md
- научная драма → ../compass/fiction/sci-drama.md
- coming-of-age → ../compass/fiction/coming-of-age.md
- драма → ../compass/fiction/drama.md
- документальный портрет → ../compass/doc/portrait.md
- cinema verité → ../compass/doc/verite.md

Проверь:
- **Хронология**: события идут в логичном порядке, нет временны́х противоречий
- **Характеры**: персонаж ведёт себя одинаково от сцены к сцене, нет немотивированных поворотов
- **Реквизит и детали**: если в сцене 3 нож, он не должен исчезнуть без объяснения
- **Жанровые нормы**: нет ли нарушений из compass-файла (например, клиффхэнгер обязателен в вертикальной микродраме)
- **Пустые сцены**: нет ли сцен без конфликта или события

Выведи список проблем по сценам:
```
Сцена 02: персонаж Марта знает о тайне, но в сцене 01 она ещё не знала → нужно объяснение
Сцена 05: нет клиффхэнгера (жанровое требование)
```

Для каждой проблемы — предложи исправление и спроси через AskUserQuestion: «Исправить?»

---

## Проход 4 — Орфография и синтаксис

Объясни перед началом:
> «Пройдусь по всем сценам: орфография, пунктуация, синтаксис. Покажу список ошибок с исправлениями, потом запишу всё сразу или по сцене — как хочешь.»

Спроси: «Начинаем?»

После подтверждения — пройди все scenes/*.md. Выведи список ошибок:

```
scenes/02_встреча.md, строка 14: «придти» → «прийти»
scenes/03_побег.md, строка 7: лишняя запятая перед «что»
```

Спроси: «Исправить всё сразу или показывать каждое изменение отдельно?»

Запиши исправленные файлы после подтверждения.

---

## Проход 5 — AI-паттерны

Объясни перед началом:
> «Проверю текст на признаки AI-письма по analytics/avoid-ai-writing-tells.md. Покажу конкретные фразы и предложу замены.»

Спроси: «Начинаем?»

Прочитай analytics/avoid-ai-writing-tells.md.

Проверь все сцены. Ключевые маркеры:
- Канцелярит: «данный», «осуществлять», «является», «в рамках», «на сегодняшний день»
- Правило трёх: ровно 3 прилагательных в ряд
- Эмоциональные плейсхолдеры: «смотрит задумчиво», «вздыхает», «многозначительно»
- Диалог объясняет чувства: «Я чувствую, что...», «Мне кажется, что...»
- Хвостовые клише: «и жизнь уже никогда не будет прежней», «это изменило всё»

Выведи список с заменами:
```
scenes/01_начало.md: «смотрит задумчиво» → удалить или заменить физическим действием
scenes/04_разговор.md: «Я чувствую, что ты уходишь» → «Ты уходишь.» (пауза)
```

Спроси через AskUserQuestion: «Применить все замены?»

---

## Финальный отчёт

После всех проходов выведи:

```
Черновик обработан.

Сцен записано: N
Файлы: scenes/01_... — scenes/NN_...
Резервная копия: versions/backup_черновик_ГГГГММДД.md

Проход 1 (структура): план из N сцен
Проход 2 (написание): N файлов создано
Проход 3 (логика): N проблем найдено, N исправлено
Проход 4 (орфография): N исправлений
Проход 5 (AI-паттерны): N замен

Следующий шаг: /compile чтобы собрать DOCX, или /analyze NN для детального разбора сцены.
```
