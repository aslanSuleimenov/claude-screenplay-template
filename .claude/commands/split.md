Read a screenplay draft and split it into scenes/blocks. Argument: $ARGUMENTS (file path).

## Шаг 1 — Определи тип проекта

Прочитай CLAUDE.md — поле **Тип** (художественный или документальный).

## Шаг 2 — Прочитай файл

Прочитай файл по указанному пути целиком. Это черновик — плоский текст, без строгого форматирования.

**Большие файлы (>50 страниц / >15000 слов):** если файл очень большой, обрабатывай по частям — сначала первую треть, потом вторую, потом третью. Создавай файлы сцен по мере обработки каждой части.

## Шаг 3 — Определи границы

### Художественный
Ищи маркеры смены сцены:
- Slug lines: ИНТ. / НАТ. / INT. / EXT. — начало новой сцены
- Заголовки типа «СЦЕНА», «Сцена», номера сцен
- Смена локации с явным указанием места
- Если маркеров нет — дели по смысловым блокам (смена места, времени, участников)

### Документальный
Ищи маркеры смены блока:
- Смена темы, героя, локации съёмки
- Переход от интервью к нарративу или наоборот
- Явные текстовые разделители (***,  ---, заголовки)
- Если маркеров нет — дели по тематическим блокам

## Шаг 4 — Создай файлы

Для каждой единицы создай файл `scenes/NN_название.md` (двузначная нумерация; если >99 единиц — трёхзначная: 001, 002...).

### Художественный формат
Строго по converter_MD_DOCX/README.md (раздел «Художественный формат»):
```markdown
# Сцена NN: Название сцены

**ИНТ./НАТ. ЛОКАЦИЯ — ВРЕМЯ СУТОК**

Описание действия.

**ИМЯ ПЕРСОНАЖА**
*(ремарка если есть)*
Текст диалога.
```

### Документальный формат
Строго по converter_MD_DOCX/README.md (раздел «Документальный формат»):
```markdown
# Блок NN: Название блока

| VIDEO | AUDIO |
|-------|-------|
| WS. Описание кадра. | **V/O:** Текст закадра. |
| CU. Лицо героя. | **SOT ИМЯ:** «Прямая речь.» |
| B-ROLL: детали. | *(NAT SOUND: описание)* |
```

## Шаг 5 — Извлеки персонажей/героев

### Художественный
Найди все имена персонажей (строки **ЖИРНЫМИ ЗАГЛАВНЫМИ** перед репликами).

### Документальный
Найди всех героев (имена после SOT, упоминания в V/O, имена в SUPER).

## Шаг 6 — Заполни CLAUDE.md

Обнови:
- **Таблицу персонажей** — имя, возраст/должность, роль
- **Таблицу сцен** — номер, файл, ключевое событие/тема
- **Структурные точки** — ключевые моменты структуры

Добавь запись в «Историю изменений».

## Шаг 7 — Итог

Выведи:
- Сколько сцен/блоков создано
- Список файлов
- Найденные персонажи/герои
- Что нужно проверить вручную
